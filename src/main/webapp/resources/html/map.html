Comparison of Google Maps on iOS, Android and the Web

This particular Javascript implementation is part of an exersize testing Web, iOS and Android versions for the purposes of comparison. Whilst the language implementations are different the development experience for iOS and Android with a few minor exceptions (background tasks and notifications) seemed to be very similar to me. There still seem to be some limitations when using the Google Maps Javascript Web API on mobile. Mainly that the event model running in a desktop browser is very different to that on mobile and it would be nice if Google attempts to address some these issues in future releases. The web HTML5 space is very exciting at the moment and if technologies like WebGL and asm.js/WebAssembly take off then I think the need to develop Apps for iOS and Android will become less relevant. All three implementations can be found on my Github repository. The transportation data used in this example was taken from the London Datastore and National Rail Enquiries and is served via KML web services on the back end of this site. The data is not being updated.
